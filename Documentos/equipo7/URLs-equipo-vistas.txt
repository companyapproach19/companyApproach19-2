URL: "/creaPedido"
	El metodo creaOrden recibe un String llamado json y devuelve otro String.
	Esta URL es para crear el pedido, recibimos el pedido en formato json (Mirar JSON inicial)
	Os devolvemos otro json que es el mismo pedido pero con el identificador

URL: "/obtenerPedido"
	El metodo obtenerOrden recibe un String llamado id y devuelve un String.
	Esta URL sirve para que dado un identificador de un pedido (int id) en formato String, 
	se devuelve el pedido asociado en formato JSON 

URL: "/pedidosNoAceptados"
	El metodo pedidosNoAceptados recibe un String llamado idActor.
	Devuelve un String que es un JSON con la lista de identificadores(mirar JSON listaIDS)
	de los pedidos no aceptados de un actor.


URL: "/pedidosEnProceso"
	El metodo pedidosEnProceso recibe un String llamado idActor.
	Devuelve un String que es un JSON con la lista de identificadores(mirar JSON listaIDS)
	de los pedidos en proceso de un actor.

URL: "/aceptarPedido"
	El metodo aceptarPedido recibe un String llamado id con la lista de pedidos a aceptar (formato JSON listaIDS)
	Devuelve un "Sucess"

URL: "/listoPedido"
	El metodo listoPedido recibe un String llamado id 
	con la lista de pedidos a poner en listo (formato JSON listaIDS)
	Devuelve un "Sucess"

En caso de dudas, consultar src/main/java/com.controller/ManejaPeticiones.java
El formato de los JSON se encuentra en Documentos/Equipo7
